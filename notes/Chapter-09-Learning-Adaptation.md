# 第 9 章：学习与适应（Learning and Adaptation）

> **一句话理解**：让 Agent 不再是静态的 — 能从经验中学习、修改自己的行为甚至代码，无需人工干预就能持续进化。从"固定程序"变成"生长中的系统"。

---

## 为什么需要它？

前面所有章节的 Agent 有一个共同特点：**你怎么写的，它就怎么跑** — 不会因为跑了 100 次而变得更好。

```
传统 Agent：第 1 次执行和第 1000 次执行行为完全一样
学习型 Agent：第 1000 次执行时已经从前 999 次中学到了更好的策略
```

## 学习方法谱系

| 方法 | 原理 | 适用场景 |
|------|------|---------|
| **强化学习** | 试错 + 奖惩反馈 | 游戏、机器人控制 |
| **监督学习** | 从标注数据学映射 | 分类、预测 |
| **无监督学习** | 在数据中发现模式 | 数据探索、聚类 |
| **少样本/零样本** | LLM 直接从示例或指令适应 | 快速任务切换 |
| **在线学习** | 持续接收新数据更新 | 实时流数据 |
| **基于记忆的学习** | 回忆过往经验调整行动 | 有历史上下文的决策 |

## 核心案例：SICA — 自我改进编码智能体

这是本章最有意思的部分 — 一个 **Agent 修改自己的源代码**来变得更好：

```
迭代循环：
  ① 审查历史版本档案 + 基准测试表现
  ② 选择表现最好的版本
  ③ 分析档案识别潜在改进点
  ④ 直接修改自己的代码库
  ⑤ 修改后的 Agent 接受基准测试
  ⑥ 结果记录在档案中
  ⑦ 重复
```

**SICA 实际改进了什么**：
- 最初：用基本的文件覆盖方法改代码
- 自我进化出"智能编辑器"：上下文相关的差异编辑
- 继续进化出"AST 符号定位器"：用语法树快速定位代码
- 再进化出"混合符号定位器"：快速搜索 + AST 结合

Agent 自己发明了更好的工具来改进自己 — 这是真正的自我进化。

## AlphaEvolve — 进化算法发现

Google 的 AlphaEvolve 把学习和适应推到了极致：

```
Gemini Flash（大量生成初始方案）
     ↓
自动化评估系统（打分）
     ↓
进化选择（保留好的，淘汰差的）
     ↓
Gemini Pro（深入分析和改进）
     ↓
循环迭代 1000+ 次
```

实际成果：
- 数据中心调度：全球计算资源使用减少 0.7%
- GPU 内核优化：FlashAttention 提速 32.5%
- 数学发现：75% 的开放问题重新发现最优解，20% 超越已知最优

## 两种对齐方法对比

| | PPO（间接） | DPO（直接） |
|--|---------|---------|
| **流程** | 先训练奖励模型 → 再用奖励模型指导 LLM | 直接用偏好数据更新 LLM |
| **复杂度** | 高（两步） | 低（一步） |
| **风险** | LLM 可能"黑入"奖励模型 | 更稳定 |
| **核心思想** | 奖励模型当裁判 | "增加偏好响应概率，减少不受欢迎响应概率" |

PPO 核心技巧：**裁剪机制** — 只允许策略小幅更新，防止灾难性遗忘。

## 本质理解

学习与适应有三个层次：

```
Level 1：参数内学习（In-context Learning）
  → 通过 prompt 中的示例"学习"，不改变模型
  → 这其实是第 8 章记忆做的事

Level 2：知识库学习（外部数据更新）
  → Agent 更新外部知识库/记忆，行为间接改变
  → RAG 知识库持续更新

Level 3：自我修改（代码/指令进化）
  → Agent 修改自己的 prompt 或代码
  → SICA、AlphaEvolve 这个层次
  → 最强大但也最危险
```

大多数实际应用停留在 Level 1-2，Level 3 目前还主要是研究领域。

## 速览（At a Glance）

- **问题背景**：Agent 在动态环境中运行，预编程逻辑不够用，没有学习能力就无法优化策略或个性化
- **解决方案**：集成学习机制将静态 Agent 转为动态演化系统，从强化学习到自我修改代码（SICA），从进化算法到 AlphaEvolve
- **实践建议**：在动态、不确定或演化环境中运行的 Agent 必须具备学习能力；对个性化和持续改进至关重要
