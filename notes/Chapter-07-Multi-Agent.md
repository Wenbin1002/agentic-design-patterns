# 第 7 章：多智能体协作（Multi-Agent Collaboration）

> **一句话理解**：前 6 章的集大成者 — 把多个专门化 Agent 组成"团队"，通过劳动分工和标准化通信，解决任何单个 Agent 无法独自完成的复杂任务。

---

## 为什么需要它？

单个 Agent 的问题：

```
用户: "帮我做一份竞品分析报告"

单 Agent 尝试：
  - 需要搜索能力（研究员）
  - 需要数据分析能力（分析师）
  - 需要写作能力（作家）
  - 需要排版能力（编辑）
  一个 Agent 什么都干 → 每项都做得一般，整体效果差

多 Agent 协作：
  研究员 Agent → 信息收集
  分析师 Agent → 数据处理和趋势识别
  作家 Agent   → 撰写报告
  编辑 Agent   → 审查润色
  各专其长，协同交付
```

---

## 5 种协作形式

| 形式 | 工作方式 | 适用场景 |
|------|---------|---------|
| **顺序交接** | A 做完 → 交给 B → B 做完 → 交给 C | 流水线式工作流 |
| **并行处理** | A、B、C 同时做不同部分 → 汇聚 | 独立子任务 |
| **辩论与共识** | 多个 Agent 持不同观点讨论 → 达成共识 | 需要多视角的决策 |
| **层次结构** | 管理者分配任务 → 工人执行 → 管理者综合 | 复杂项目管理 |
| **批评者-审查者** | 生产者生成 → 多个审查者多角度评估 → 修订 | 质量要求高的内容 |

---

## 6 种组织模型

书中给出了从简单到复杂的 6 种 Agent 组织模型：

```
1. 单智能体      ── 无交互，自主运行
2. 网络          ── 去中心化点对点通信
3. 监督者        ── 中央协调者指挥下属
4. 监督者作为工具 ── 监督者提供资源/指导而非命令
5. 层次化        ── 多层监督结构（监督者的监督者）
6. 自定义        ── 根据需求混合上述模型
```

**选择依据**：

| 因素 | 偏向简单模型 | 偏向复杂模型 |
|------|------------|------------|
| 任务复杂度 | 低 | 高 |
| Agent 数量 | 少（2-3） | 多（5+） |
| 自主程度需求 | 低 | 高 |
| 容错要求 | 低 | 高（单点故障不可接受） |
| 通信开销容忍度 | 低 | 高 |

## 代码示例要点

### CrewAI 方式 — 角色 + 任务 + 团队

```python
# 定义专门角色
researcher = Agent(role='高级研究分析师', goal='查找并总结AI趋势')
writer = Agent(role='技术内容作家', goal='撰写引人入胜的博客文章')

# 定义任务及依赖关系
research_task = Task(description="研究前3个AI趋势", agent=researcher)
writing_task = Task(description="撰写500字博客", agent=writer, 
                    context=[research_task])  # 依赖研究结果

# 组建团队
crew = Crew(agents=[researcher, writer], tasks=[research_task, writing_task],
            process=Process.sequential)
crew.kickoff()
```

核心抽象：**Agent（谁做）+ Task（做什么）+ Crew（怎么协调）**

### Google ADK 方式 — 层次化 + Agent 作为工具

```python
# 层次结构：Coordinator 委托给子智能体
coordinator = LlmAgent(
    name="Coordinator",
    instruction="欢迎时委托给 Greeter，任务时委托给 TaskExecutor。",
    sub_agents=[greeter, task_doer]  # 自动建立父子关系
)

# Agent 作为工具：一个 Agent 把另一个 Agent 当工具用
image_tool = AgentTool(agent=image_generator_agent)
artist_agent = LlmAgent(
    instruction="先构思创意提示词，然后用 ImageGen 工具生成图像。",
    tools=[image_tool]  # 另一个 Agent 被包装成了工具！
)
```

核心洞察：**"工具"可以是另一个 Agent** — 模糊了工具使用和多智能体协作的边界。

### ADK 编排原语

| 原语 | 作用 | 类比 |
|------|------|------|
| `SequentialAgent` | 子 Agent 顺序执行 | 流水线 |
| `ParallelAgent` | 子 Agent 并发执行 | 多线程 |
| `LoopAgent` | 子 Agent 循环执行直到条件满足 | while 循环 |
| `AgentTool` | 把 Agent 包装成工具给其他 Agent 调用 | 函数调用 |

## 本质理解

多智能体协作的本质是**人类团队协作模式在 AI 系统中的复刻**：

```
人类团队                    多智能体系统
─────────                  ──────────────
项目经理分配任务     →      协调者 Agent 委托子任务
各部门专家各司其职   →      专门化 Agent 执行子任务
团队会议沟通进展     →      Agent 间状态共享和消息传递
质量审核和评审       →      批评者/审查者 Agent
最终交付整合         →      综合 Agent 汇总结果
```

## 速览（At a Glance）

- **问题背景**：单个 Agent 缺乏解决多方面任务的全部技能，造成瓶颈和次优结果
- **解决方案**：将问题分解为子任务，分配给专门化 Agent，通过通信协议（顺序/并行/层次/辩论）协作，产生协同效应
- **实践建议**：当任务可分解为需要不同专业技能的子任务时使用；适合研究分析、软件开发、创意内容等复杂任务
